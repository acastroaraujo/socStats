12.1 Generalized Linear Models

GLMs are a very general framework that allows us to model different types of outcome variables (e.g., binary, count, continuous).

They all involve the following components:

  • A vector of outcome data \(\boldsymbol y = (y_1, y_2, \dots, y_n)\)

  • A probability distribution for \(\boldsymbol y\)â€”e.g., Binomial, Multinomial, Poisson, Normal, etc.

  • A matrix of predictors \(\mathbf X\) and a vector of coefficients \(\boldsymbol \beta\), that together form the linear predictor \(\mathbf X \boldsymbol \beta\).

  • A link function \(g\) that maps the linear predictor to the outcome data via the chosen probability distribution.

Taken together, we have the following:

\[ \begin{align} \overset{\text{Linear Predictor}}{\boldsymbol{\eta} = \mathbf{X} \boldsymbol{\beta}} &&\overset{\text{Link Function}}{\boldsymbol \eta =g(\boldsymbol \theta)} &&\overset{\text{Probability Distribution}}{\boldsymbol y \sim f_Y(\boldsymbol \theta, ...)} \end{align} \]

Note. The reason I chose \(f_Y\) to denote probability distribution is because if the data is modeled as continuous (e.g., with a Normal distribution), then \(f_Y\) will be a probability density function.

12.1.1 Logistic

Outcome Type Distribution Link Inverse Link
Binary \(\small Y \sim \text{Bernoulli}(p)\) Logit Inverse Logit
\[ Y \in \{0, 1\} \] \[ \small \Pr(Y = y) = \begin{cases} 1- p &\text{if } \ y = 0 \\ p &\text{if } \ y = 1 \\ 0 &\text{otherwise} \end{cases} \] \[ \log\bigg(\frac{p}{1-p}\bigg) = \mathbf X \boldsymbol \beta \] \[ p = \frac{e^{\mathbf X \boldsymbol \beta}}{1 + e^{\mathbf X \boldsymbol \beta}} \]

Formula Syntax

glm(y ~ x1 + x2 + ..., family = binomial(link = "logit"))

Coefficients

For a 1 unit increase in \(x\), we expect the log-odds of \(y\) to increase by \(\beta\).

Example

12.1.2 Poisson

Outcome Type Distribution Link Inverse Link
Counts (Integers) \(\small Y \sim \text{Poisson}(\lambda)\) Log Exponent
\[ Y \in \{0, 1, 2, ...\} \] \[ \small \Pr(Y = y) = \frac{\lambda^ye^{-\lambda}}{y!} \] \[ \log(\lambda) = \mathbf X \boldsymbol \beta \] \[ \lambda = e^{\mathbf X \boldsymbol \beta} \]

Formula Syntax

glm(y ~ x1 + x2 + ..., family = poisson(link = "log"))

Coefficients

For a 1 unit increase in \(x\), we expect the log of \(y\) to increase by \(\beta\).

Example

Suppose we want to model traffic accidents, \(i\) could index street intersections in a city and \(y_i\) could be the number of traffic accidents at intersection \(i\) in a given year. We include two predictors: mph measures the average speed of traffic near the intersection and signal is an indicator for whether the intersection has a traffic signal or not.

After estimating the model via maximum likelihood estimate, we get the following:

\[ \begin{align} y_i &\sim \text{Poisson}(\lambda_i) \\ \log(\lambda_i) &= 2.8 + 0.012 \cdot \text{mph}_i - 0.12 \cdot \text{signal}_i \end{align} \]

Remeber that \(\lambda\) is the expected number of counts. If we exponentiate both sides, we get the following expression:

\[ \begin{align} \lambda_i &= e^{(2.8 + 0.012 \cdot \text{mph}_i - 0.12 \cdot \text{signal}_i)} \\ &= e^{(2.8)} e^{ (0.012 \cdot \text{mph}_i)} e^{(- 0.12 \cdot \text{signal}_i)} \end{align} \]

We can then interpret each coefficient as follows:

  • The constant term gives the intercept of the regression, that is, the prediction if \(\text{mph}_i = 0\) and \(\text{signal}_i = 0\). Since this is not possible (no street will have an average traffic speed of 0), we will not try to interpret the constant term.

  • The coefficient of \(\text{mph}_i\) is the expected difference in y (on the logarithmic scale) for each additional mph of traffic speed. Thus, the expected multiplicative increase is \(e^{0.012} = 1.012\) , or a 1.2% positive difference in the rate of traffic accidents per mph. Since traffic speeds vary by tens of mph, it would make sense to define \(\text{mph}_i\) as speed in tens of mph, in which case its coefficient would be 0.12, corresponding to a 12% increase (more precisely, \(e^{0.12} = 1.127\): a 12.7% increase) in accident rate per ten mph.

  • The coefficient of \(\text{signal}_i\) tells us that the predictive difference of having a traffic signal can be found by multiplying the accident rate by \(e^{âˆ’0.20} = 0.82\) yielding a reduction of 18%.

As usual, each regression coefficient is interpreted as a comparison in which one predictor differs by one unit while all the other predictors remain at the same level, which is not necessarily the most appropriate assumption when extending the model to new settings. For example, installing traffic signals in all the intersections in the city would not necessarily be expected to reduce accidents by 18%.

Gelman et al. (2020, p. 267)

12.1.3 Gaussian (Normal)

Outcome Type Distribution Link Inverse Link
Continuous \(\small Y \sim \text{Normal}(\mu, \sigma^2)\) Identity Identity
\[ Y \in (-\infty, + \infty) \] \[ f(y) = \frac{1}{\sigma \sqrt{2 \pi}} e^{- \frac{1}{2} \big(\frac{y - \mu}{\sigma}\big)^2} \] \[ \mu = \mathbf X \boldsymbol \beta \] \[ \mu = \mathbf X \boldsymbol \beta \]

Model Syntax in R

glm(y ~ x1 + x2 + ..., family = gaussian(link = "identity"))

Coefficients

For a 1 unit increase in \(x\), we expect \(y\) to increase by \(\beta\).

Farewell to epsilon

Some of you might be familiar with an alternative notation for the Gaussian linear model, in which the â€œerror termâ€ is part of the equation linking the linear predictor to the outcome variable.

\[ \begin{align} y_i &= \alpha + \beta_1 x_{1i} + \beta_2 x_{2i} + \varepsilon_i, \\ \varepsilon_i &\sim \text{Normal}(0, \sigma) \end{align} \]

This is equivalent to the more general notation used above.

\[ \begin{align} y_i &\sim \text{Normal}(\mu, \sigma), \\ \mu &= \beta_0 + \beta_1 x_i \end{align} \]

Since both are equivalent, you can use whichever you want. However, it does not make sense to include the â€œerror termâ€ in other types of models like logistic or Poisson.

12.1.6 OLS

If youâ€™ve seen linear regression before, you probably started with the Gaussian linear model and estimated the parameters using something called â€œordinary least squaresâ€ instead of â€œmaximum likelihood.â€

These approaches give equivalent answers.

Ordinary Least Squares Maximum Likelihood
Find parameters that minimize the sum of squared residuals. Find parameters that maximize the log-likelihood.

The reason they give equivalent answers is because the log-likelihood is maximized when the sum of \((y_i - \mu_i)^2\) is minimized.

## Generalized Linear Models

GLMs are a very general framework that allows us to model different types of outcome variables (e.g., binary, count, continuous).

They all involve the following components:

-   A vector of outcome data $\boldsymbol y = (y_1, y_2, \dots, y_n)$

-   A probability distribution for $\boldsymbol y$---e.g., Binomial, Multinomial, Poisson, Normal, etc.

-   A matrix of predictors $\mathbf X$ and a vector of coefficients $\boldsymbol \beta$, that together form the *linear predictor* $\mathbf X \boldsymbol \beta$.

-   A *link function* $g$ that maps the linear predictor to the outcome data via the chosen probability distribution.

Taken together, we have the following:

$$
\begin{align}
\overset{\text{Linear Predictor}}{\boldsymbol{\eta} = \mathbf{X} \boldsymbol{\beta}} 
&&\overset{\text{Link Function}}{\boldsymbol \eta =g(\boldsymbol \theta)}
&&\overset{\text{Probability Distribution}}{\boldsymbol y \sim f_Y(\boldsymbol \theta, ...)}
\end{align}
$$

*Note. The reason I chose* $f_Y$ *to denote probability distribution is because if the data is modeled as continuous (e.g., with a Normal distribution), then* $f_Y$ *will be a probability **density** function.*

**Coefficients**

For a 1 unit increase in $x$, we expect the log of $y$ to increase by $\beta$.

**Example**

Suppose we want to model traffic accidents, $i$ could index street intersections in a city and $y_i$ could be the number of traffic accidents at intersection $i$ in a given year. We include two predictors: `mph` measures the average speed of traffic near the intersection and `signal` is an indicator for whether the intersection has a traffic signal or not.

After estimating the model via maximum likelihood estimate, we get the following:

$$
\begin{align}
y_i &\sim \text{Poisson}(\lambda_i) \\
\log(\lambda_i) &= 2.8 + 0.012 \cdot \text{mph}_i - 0.12 \cdot \text{signal}_i 
\end{align}
$$

Remeber that $\lambda$ is the expected number of counts. If we exponentiate both sides, we get the following expression:

$$
\begin{align}
\lambda_i &= e^{(2.8 + 0.012 \cdot \text{mph}_i - 0.12 \cdot \text{signal}_i)} \\
&= e^{(2.8)} e^{ (0.012 \cdot \text{mph}_i)} e^{(- 0.12 \cdot \text{signal}_i)}
\end{align}
$$

> We can then interpret each coefficient as follows:
>
> -   The constant term gives the intercept of the regression, that is, the prediction if $\text{mph}_i = 0$ and $\text{signal}_i = 0$. Since this is not possible (no street will have an average traffic speed of 0), we will not try to interpret the constant term.
>
> -   The coefficient of $\text{mph}_i$ is the expected difference in y (on the logarithmic scale) for each additional mph of traffic speed. Thus, the expected multiplicative increase is $e^{0.012} = 1.012$ , or a 1.2% positive difference in the rate of traffic accidents per mph. Since traffic speeds vary by tens of mph, it would make sense to define $\text{mph}_i$ as speed in tens of mph, in which case its coefficient would be 0.12, corresponding to a 12% increase (more precisely, $e^{0.12} = 1.127$: a 12.7% increase) in accident rate per ten mph.
>
> -   The coefficient of $\text{signal}_i$ tells us that the predictive difference of having a traffic signal can be found by multiplying the accident rate by $e^{âˆ’0.20} = 0.82$ yielding a reduction of 18%.
>
> As usual, each regression coefficient is interpreted as a comparison in which one predictor differs by one unit while all the other predictors remain at the same level, which is not necessarily the most appropriate assumption when extending the model to new settings. For example, installing traffic signals in all the intersections in the city would not necessarily be expected to reduce accidents by 18%.
>

**Farewell to epsilon**

Some of you might be familiar with an alternative notation for the Gaussian linear model, in which the "error term" is part of the equation linking the linear predictor to the outcome variable.

$$
\begin{align}
y_i &= \alpha + \beta_1 x_{1i} + \beta_2 x_{2i} + \varepsilon_i, \\
\varepsilon_i &\sim \text{Normal}(0, \sigma)
\end{align}
$$

This is equivalent to the more general notation used above.

$$
\begin{align}
y_i &\sim \text{Normal}(\mu, \sigma), \\
\mu &= \beta_0 + \beta_1 x_i
\end{align}
$$

Since both are equivalent, you can use whichever you want. However, it does not make sense to include the "error term" in other types of models like logistic or Poisson.

### OLS

If you've seen linear regression before, you probably started with the Gaussian linear model and estimated the parameters using something called "ordinary least squares" instead of "maximum likelihood."

These approaches give equivalent answers.

| Ordinary Least Squares                                      | Maximum Likelihood                                |
|---------------------------------------|---------------------------------|
| Find parameters that minimize the sum of squared residuals. | Find parameters that maximize the log-likelihood. |

The reason they give equivalent answers is because the log-likelihood is maximized when the sum of $(y_i - \mu_i)^2$ is minimized.

In order to assess goodness of fit, it is sometimes useful to plot the residuals against some predictor.

$$

$$

If the model fits well, we should expect to see the residuals to be normally distributed around zero.

  geom_hline(yintercept = 0, linetype = "dashed") +
